# Exp - 6 STUDENT ONLINE EXAM REGISTRATION SYSTEM

# AIM:To develop the Exam Registration System using Star UML tools.

# SRS (Procedure):
```
Problem Identification – Identify the need to conduct exams online efficiently and securely.
Requirement Gathering – Collect requirements from students, teachers, and administrators.
System Analysis – Study existing manual or offline exam methods and identify issues like time, errors, and security.
System Design – Design modules for question management, exam scheduling, student login, and result processing.
Functional Specification – Define main functions like add questions, conduct exams, auto-evaluate answers, and generate results.
Non-Functional Specification – Specify requirements for performance, security, reliability, and user-friendliness.
Testing & Validation – Test all modules to ensure correct question display, time limits, and result accuracy.
Documentation & Maintenance – Prepare reports, user guides, and update the system as per new exam needs.
```
